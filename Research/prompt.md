# SwiftSeat Application Prompts

These prompts include full project context and requirements. Paste them directly into the Stitch web interface to generate each application.

## 1. User Application (Mobile)
**Project Context:**
Generate a mobile application interface for **SwiftSeat**, a Smart Venue Experience Assistant designed to revolutionize high-density stadium events. SwiftSeat uses Google Gemini AI and a predictive routing algorithm (PALO) to eliminate crowd congestion and optimize concession wait times.

**What's Needed:**
- **AI Concierge Interface**: A central conversational UI for users to place food orders and ask for directions.
- **Stadium Map & Routing**: A map view displaying a real-time crowd density heatmap and intelligent routing to concessions.
- **Digital Wallet**: A section for displaying digital tickets and concession vouchers.
- **Accessibility features**: High-contrast "Calm UI" for glare, WCAG 2.1 AA compliance, and mobility-friendly routing toggles.
- **Emergency State**: UI must be capable of stripping all commercial elements to show a singular escape vector during emergencies.

**Aesthetic:**
Fully polished, professional, and publish-ready. The UI/UX must be stunning, visually engaging, and highly modern. Focus heavily on smooth micro-interactions and state-of-the-art mobile design.

## 2. Vendor Portal (Web Application)
**Project Context:**
Generate a web application dashboard for concession vendors operating within the **SwiftSeat** stadium ecosystem. The venue receives thousands of AI-routed orders dynamically balanced by our PALO algorithm.

**What's Needed:**
- **Order Queue Management**: Real-time incoming order feeds placed via the stadium's AI concierge.
- **Wait-Time Metrics**: Visual indicators of current queue depth and load-balancing parameters.
- **Inventory & Status**: Toggles to update item availability or mark orders as ready for pickup.

**Aesthetic:**
Highly functional for administrative usage but maintaining an excellent, data-rich design. The layout must handle large data tables, live status dashboards, and metrics efficiently. The visual style should be sleek, clean, and extremely visually appealing, balancing dense information with exceptional usability.

## 3. Admin Portal (Web Application)
**Project Context:**
Generate a comprehensive web application for stadium system administrators to monitor the entire **SwiftSeat** smart venue ecosystem. The admin controls crowd flow, safety alerts, and global metrics.

**What's Needed:**
- **Global Density Heatmap**: A massive real-time visualization of the concourse, tracking 50,000+ attendees and identifying congestion points.
- **Emergency Broadcast Controls**: Interface for triggering dynamic density alerts and escape routing to user devices.
- **Telemetry & Algorithm Controls**: Dashboards showing live data from the backend mock generator and controls to tweak PALO routing penalties.
- **System Health Grid**: Comprehensive data grids showing wait times and loads across all vendors and stadium sections.

**Aesthetic:**
Built to support heavy administrative tasks and present extensive data clearly. Emphasize high-end data visualization, complex data grids, and robust control panels. The design must be extremely nice to the eye, employing modern web aesthetics, premium dark/light themes, and an organized layout suitable for extended professional use by security and operations staff.
