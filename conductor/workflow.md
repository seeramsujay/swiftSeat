# Workflow: SwiftSeat

## Coding Standards
1. **Component-First**: Modular components with isolated logic in hooks.
2. **Accessibility**: WCAG 2.1 AA compliance (high contrast, ARIA).
3. **Type Safety**: Mandatory Interfaces for Props and Data models.
4. **Style Guide**: Follow "The Vigilant Lens" (Admin) and corresponding role guidelines.

## Process
1. **Context Check**: Verify `conductor/` artifacts.
2. **Implementation Plan**: Update plan for major features.
3. **Task Tracking**: Use `task.md` for execution steps.
4. **Verification**: Lint, Build, and Functional test against local backend.

## Naming Conventions
- Components: `PascalCase` (`UserConcierge.tsx`)
- Hooks: `camelCase` (`usePALO.ts`)
- Styles: `kebab-case` (`layout-shell.css`)
